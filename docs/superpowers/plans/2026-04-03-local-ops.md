# Local Ops Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** implement local operational tooling for `my-agent-harness` so staged installs can be listed, diagnosed for drift, and repaired without touching `~/.claude` or `~/.codex`.

**Architecture:** a shared ops helper under `scripts/lib/` owns state loading, target-root safety checks, digest recomputation, and drift comparison. Thin entrypoint scripts (`list-installed.sh`, `doctor.sh`, `repair.sh`) reuse that helper, and `repair.sh` delegates rebuilding to the existing sync scripts.

**Tech Stack:** Bash, `jq`, staged filesystem inspection, shell integration tests
